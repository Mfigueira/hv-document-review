# 03 � Design (Sketches)

**Decision:** we build **Option A � Split View** (below). Options B and C are kept
at the end as the alternatives considered (useful design-thinking context for the
write-up). All wireframes reflect the locked interaction model: **checkbox**
issue resolution, a **context-dependent CTA** (Re-upload vs Submit), and a
**Submitted Page**.

> These are low-fidelity ASCII wireframes for fast iteration. Higher-fidelity
> image mockups can come later if useful for the presentation.

## Shared elements (all options)

- **Header:** product name ("Document Review") � file name � `v{version}` �
  status badge (`On review`) � assignee (Jane Cooper).
- **Submission status:** a persistent bar stating exactly what blocks the action,
  e.g. **"8 of 12 blocking issues resolved � 4 left before re-upload"**, plus the
  context-dependent CTA.
- **Issue checkboxes:** every issue has a checkbox; only **critical + major**
  gate the CTA. Checked state persists across reloads (`localStorage`).
- **CTA logic:**
  - Critical/major present ? **`Re-upload`**, disabled until all are checked.
  - Only minor (or none) ? **`Submit`**, always enabled ? Submitted Page.
- **Severity system:** consistent color + icon + label for critical / major / minor.
- **Footer:** minimal � attribution / version.
- **Dev panel:** small floating control to switch demo scenarios.

Legend: `?` PDF page � `?` critical � `?` major � `?` minor � `[?]`/`[ ]` checkbox � `[ ]` button.

## Visual / design tokens (Tailwind theme)

- **Severity palette:** critical = red, major = amber/orange, minor = slate/blue-grey.
- **Status badge:** `on_review` = blue; `submitted` = green.
- **CTA states:** primary accent enabled; clearly muted + tooltip ("Resolve all
  critical & major issues first") when disabled.
- Generous whitespace, one accent color for primary actions.
- Positive cleared state: bar turns green with the matching CTA.
